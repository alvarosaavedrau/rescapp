Change Gnu/Linux password - Rescapp - Documentation
  • Step 1

    Rescapp searches for partitions that have on them the following file: /etc/issue based on information found in the /proc/partitions file.
  • Step 2

    If such partitions have been found, Rescapp displays partitions that have on them the following file: /etc/issue. At the same time you are prompted:

    Which GNU/Linux partition?

    You are supposed to choose your main GNU/Linux distribution (you will probably only see one) and press the OK button.

  • Step 3

    You are prompted:

    Which user?

    Select the desired user from the list.

  • Step 4

    You are prompted:

    Enter the password (user)

    Use a temporary password please. You will change it later in your system.

    • Step 5

    Rescapp tries to change the password for the selected user.
  • Step 6

    You are informed about the password change process.

    You either get:

    [SUCCESS] The Password was changed OK! :). We recommend that you change it once you're back in your system to use your systems default encryption

    or

    [ERROR] The password was not changed. Something went wrong! :(

  • Step 7

    A log file called chpasswd.log.txt is generated so that you can share it later if needed.
