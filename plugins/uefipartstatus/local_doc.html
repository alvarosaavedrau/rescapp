Check UEFI partition status - Rescapp - Documentation

Newest computers start up thanks to the newest (from 2015 onwards) UEFI boot system.

If you want to switch from an old BIOS system to the newest UEFI system you need an EFI partition.

But the EFI partition needs to crafted in an very specific way. This option let's you check if your EFI partition has been crafted properly.

Please note that in some UEFI systems you might need to press an special hotkey (such as F8 or F10) in order to show up the UEFI boot menu in order to be able to choose your added UEFI boot entry.

  • Step 1

    Rescapp searches for partitions on your system.
  • Step 2

    If such partitions have been found, Rescapp displays them. At the same time you are prompted:

    Which EFI partition?

    You are supposed to choose, your EFI System partition and press the OK button.

  • Step 3

    You are informed about the EFI partition check by fdisk.

    You either get:

    [SUCCESS] The EFI partition, has been detected, as an EFI partition by fdisk :)

    or

    [ERROR] The EFI partition, has NOT been detected, as an EFI partition by fdisk. :(

  • Step 4

    You are informed about the esp flag check.

    You either get:

    [SUCCESS] The EFI partition, has a esp flag :)

    or

    [ERROR] The EFI partition, does not have a esp flag. :(

  • Step 5

    You are informed about the boot flag check.

    You either get:

    [SUCCESS] The EFI partition, has a boot flag :)

    or

    [ERROR] The EFI partition, does not have a boot flag. :(

  • Step 6

    You are informed about the valid uefi filesystem check.

    You either get:

    [SUCCESS] The EFI partition, has a valid uefi filesystem :)

    or

    [ERROR] The EFI partition, does not have a valid uefi filesystem. :(

  • Step 7

    You are informed about the partition mount check.

    You either get:

    [SUCCESS] The EFI partition, can be mounted :)

    or

    [ERROR] The EFI partition, cannot be mounted. :(

  • Step 8

    You are informed about the disk type check.

    You either get:

    [SUCCESS] The EFI partition, disk type is either gpt or msdos :)

    or

    [ERROR] The EFI partition, disk type is neither gpt nor msdos. :(

  • Step 9

    You are informed about the overall Check UEFI partition status process.

    You either get:

    [SUCCESS] The EFI partition status was OK (Every test passed successfully.) ! :)

    or

    [ERROR] This might not be a valid EFI partition. At least one test failed. :(
